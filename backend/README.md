# MCI / Control Classifier — FastAPI Backend

Accepts `.cha` transcript files via API upload and returns a **Control** or **MCI**
prediction using a trained Random Forest model.

## Quick Start (local)

```bash
cd backend
pip install -r requirements.txt
uvicorn main:app --reload



python -m uvicorn main:app --host 127.0.0.1 --port 8000
```

Server starts at **http://127.0.0.1:8000**  
Interactive docs at **http://127.0.0.1:8000/docs**

## API Endpoints

| Method | Path              | Description                        |
|--------|-------------------|------------------------------------|
| GET    | `/health`         | Health check & model info          |
| POST   | `/predict`        | Upload **one** `.cha` file         |
| POST   | `/predict/batch`  | Upload **multiple** `.cha` files   |
| POST   | `/predict/multimodal` | Upload `.cha` + MRI image for fused 4-class prediction |
| POST   | `/predict/cnn-cha-fusion` | Upload `.cha` + MRI image for fused binary Control/MCI prediction |

### Example — cURL

```bash
curl -X POST http://127.0.0.1:8000/predict \
  -F "file=@/path/to/patient.cha"
```

### Example — cURL (multimodal)

```bash
curl -X POST http://127.0.0.1:8000/predict/multimodal \
  -F "cha_file=@/path/to/patient.cha" \
  -F "mri_file=@/path/to/mri.jpg"
```

### Example — cURL (CNN+CHA binary)

```bash
curl -X POST http://127.0.0.1:8000/predict/cnn-cha-fusion \
  -F "cha_file=@/path/to/patient.cha" \
  -F "mri_file=@/path/to/mri.jpg"
```

Optional threshold override:

```bash
curl -X POST http://127.0.0.1:8000/predict/cnn-cha-fusion \
  -F "cha_file=@/path/to/patient.cha" \
  -F "mri_file=@/path/to/mri.jpg" \
  -F "threshold_override=0.42"
```

### Example — Python requests

```python
import requests

with open("patient.cha", "rb") as f:
    r = requests.post("http://127.0.0.1:8000/predict", files={"file": f})

print(r.json())
```

## Model Artifacts

Place these three files in `backend/model_artifacts/`:

| File | Description |
|------|-------------|
| `ad_mci_model.pkl` | Trained sklearn classifier |
| `feature_scaler.pkl` | Fitted StandardScaler |
| `feature_names.pkl` | Ordered list of feature column names |

They are generated by the training notebook (`random_forest.ipynb`, Step 17).

## Multimodal Artifacts (MRI + Voice)

For `/predict/multimodal`, place these files in `backend/model_artifacts/multimodal/`:

| File | Description |
|------|-------------|
| `multimodal_rf_model.pkl` | Trained fusion Random Forest pipeline |
| `multimodal_feature_names.pkl` | Ordered feature list used by fusion model |

MRI image feature extraction in the endpoint also needs a Keras MRI model (`.h5`).
By default, backend looks at `../Alzheimers-Disease-Classification/Somesh_VGG16.h5`.
Set `MRI_MODEL_PATH` to override.

## CNN+CHA Fusion Artifacts (for `/predict/cnn-cha-fusion`)

Place these files in:

- `backend/model_artifacts/cnn_cha_fusion/`

Required:

- `cnn_cha_fusion_v2.keras`
- `voice_scaler.pkl`
- `voice_feature_columns.pkl`

Optional:

- `cnn_cha_fusion_v2_report.json` (reads `best_threshold`; defaults to `0.5`)

If files are not found in backend folder, service falls back to:

- `../cnn_cha_fusion/artifacts/`

Override with env var:

- `CNN_CHA_ARTIFACT_DIR`
